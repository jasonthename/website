About Me

I have always been interested in computers, but I didn't really think much of it until I created my first program. My Projects

A little while ago I started working on a few Python projects. They were larger than the other programs that I was used to creating for so long, but I definitely feel like I've taken a step in the right direction. Most of these projects can be found on my GitHub profile. I initially created my GitHub account in order to participate in a weekend-long code jam - it was actually really fun - and since then I've used it to collaborate with other programmers to work on some pretty cool projects! I've put a couple of them here for now.

Maze Solver

This program does exactly what you think - it solves mazes. It uses a depth-first search (DFS) algorithm alongside dynamic dead-end filling in order to efficiently find it's way to the end of the maze. The only improvement I would make is allow it to use images which aren't specially formatted. I imagine this would require the use of machine learning to achieve consistent results, but at the moment I'm not quite at that level. Maybe I could be in the future!

Intel 8008 Simulator

This one is one of my favourites. I've been working with fellow programmer Joseph Banks to create a program which simulates Intel's 8008 microprocessor. So far we've created the registers, the memory and a lot of the internal bitwise arithmetic operators, but it's very much still a work in progress. We plan to have it execute the same assembly code that an actual 8008 processor would, and hopefully we will reach that as our end goal.


This Website

I've mostly enjoyed developing the back-end of this website, because it's written in Python with the Flask framework. It's all neatly wrapped into one application which is deployed in a simple Docker container. My newest addition to the website is a message corner (but it consists of more nonsense than actual messages*). It uses an SQLite database to store each message, and POST requests are rate-limited so that some attacks can be prevented, though I am considering a more defensive approach such as a CAPTCHA to give it a very reliable anti-botting system. Some fellow developers decided to test my input sanitisation by attempting to inject various HTML, SQL and JS into the application, none of which caused any harm. I don't particularly mind people doing that, as I decided to do this project mostly to learn how to reliably handle forms and databases with Flask anyways. If someone finds a weak spot, it would just be another thing to learn and apply for me. You can find all the code for my website on GitHub
*I created an administrator panel so I can easily manage messages if they get out of hand, of course. I'm always prepared!

I'm also thinking about working on many other projects in the near future. Although I have already made one contribution to a larger community-run project, I do plan on doing a lot more for a variety of different repositories, because I feel like it would be a great learning experience to work alongside a group of others who have similar interests.
